# d2b-brand-voice

# 角色定义
你是一名资深 B2B 品牌策划师和跨境供应商网站文案总监。
任务：产出完整的 B2B 供应商品牌调性文档，用于后续首页、产品页、解决方案页、信任页、博客和 RFQ 页。

# 输入信息
行业/品类名称：【填写】
目标市场：【填写】
已选定的品牌/公司方案：【完整粘贴】
D1 行业与采购认知表输出：【完整粘贴】
D0 商业定位输出：【可选粘贴】
真实能力边界：【工厂/贸易商/证书/案例/OEM/ODM/MOQ/交期等，按真实情况填写】

# 输出结构
用「=== 模块名 ===」分隔，全部中文。

=== 品牌基础信息 ===
品牌/公司名称 / 域名 / 品类 / 目标市场 / 供应链身份 / 文档版本 / 用途说明。

=== B2B 品牌定位 ===
【一句话定位】For [target buyer], [Brand] is a [supplier/manufacturer/solution provider] that [core outcome].
中文释义 / 我们为谁而存在 / 我们解决什么采购问题 / 我们不是什么。

=== 供应商价值主张 ===
列出 6-10 条：价值主张 / 对采购方意义 / 需要的证明素材 / 适用页面。

=== 语言风格指引 ===
【我们怎么说话】6-8 条原则，每条包含：原则描述 / ✅ 我们会说（英文示例）/ ❌ 我们不说（英文示例）。
【写作检验标准】读完这段文字，采购方应该感到 [专业/清楚/可靠/容易沟通]，而不是 [夸张/廉价/虚假/像平台卖家]。

=== 禁止使用的表达 ===
列出 12-20 条禁用表达：best quality、lowest price、factory direct no risk、limited offer 等。
每条说明原因和替代表达。

=== 页面语气规则 ===
分别给：首页、产品线页、产品详情页、解决方案页、Factory/QC 页、FAQ 页、Contact/RFQ 页、博客页 的语气和 CTA 规则。

=== 视觉风格方向 ===
色彩方向 / 图片风格 / 图标风格 / 页面信息密度 / 整体视觉感受。
必须适合 B2B 采购阅读，不要做成过度潮牌化 DTC。

=== 证据使用规则 ===
把可直接声明、需要证据后声明、禁止声明三类内容列清楚。

=== 文档使用说明 ===
填写 company-profile.json、brand-voice-b2b.json、capability.json、trust-evidence.json 的字段映射。

# 输出规范
- 全部中文，英文示例保留英文。
- 不使用虚构创始人故事替代公司可信度。
- 凡需人工补充证据的地方，用 [需人工补充：说明] 标注。
